- ORM converts Python objects into database records - We don't write SQL queries directly - Flask-SQLAlchemy provides ORM for Flask What is Flask-Migrate? - Manages database migrations (schema changes) - Tracks database version history - Creates migration files automatically 1)The database will use app.py after running this command :-set FLASK_APP = app.py then we will initialize it by this command: flask db init (creates migrations folder) and then we will migrate with this command :-flask db migrate -m "Initial migration" (creates a migration file) and finally we will upgrade the database with this command :-flask db upgrade (applies migration to database) Why we do this: - set FLASK_APP = app.py: Tells Flask which file is the main app file - flask db init: Creates migrations folder to track database changes - flask db migrate -m "Initial migration": Creates a migration file that shows what changes need to be made to database schema - flask db upgrade: Applies those changes to the actual database // DATABASE (MySQL Community) connections 1)mysql pymysql URL: type on Google to find the URL.
